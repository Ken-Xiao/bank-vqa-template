# 写一份董事会深度分析汇报 · RSM 咨询深度版

> 让董事会读完不用追问，下个季度知道批准什么。

## Why this skill works

- **独立阅读型版式**：每页主标 H1、副标 H2、数据陈述、来源条四层完整，董事邮件分发直接读完，不依赖讲者口述。
- **SCQA 先抑后扬节拍**：账面好 → 实质偏弱 → 市场已定价 → 行动方案——读完第 5 页董事会会问"那我们该做什么"，而不是"凭什么"。
- **双视角交叉验证骨架**：经营视角 + 资本市场 / 行业实证视角并列，把"顾问主观判断"升级为"两个独立证据源指向同一结论"。

## Methodology cheat-sheet

**SCQA + 金字塔混合叙事** — 先用 Situation → Complication → Question 建立张力（前 1/3），再用金字塔结构展开 Answer（后 2/3）。

四个节拍对应 9 模块骨架（按 ~46 页报告分配）：

1. **S 情境**（开篇 3 页 + 摘要 1 页）：让董事认识被分析的对象、研究口径、4 项核心指标全景。
2. **C 复杂化**（北极星结论 1 页 + 方法论 2-3 页）：明确"账面 ≠ 价值"或对应张力，引入分析框架。
3. **Q 问题**（行业坐标 + 4 个专题模块 + 实证模块，共 ~25 页）：用同业对标 + 计量证据回答"差距究竟在哪"。
4. **A 答案**（三大矛盾 + 行动地图 + 闭幕，共 ~10 页）：从矛盾收敛到 12 个月行动 + 三套压力情景 + 两种未来图景。

**语言纪律 4 条**（必须遵守）：
- ❌ 比喻、文学化用语（"含金量""红利""定价悬崖""失分""命中"等）
- ❌ 带主观倾向的评价词（"薄弱""偏弱""遗憾"等）
- ✅ 全文中文为主；英文术语在首次出现时以"中文（英文缩写）"形式注解一次
- ✅ 每页右下数据来源条 + 客观陈述句式（"低于中位 X bp""排名第 N"）

## Before / After

### 模块小结页（每个主题段结束的总结页）

> **典型 PPT 模板**：
> 用一页放 3-5 个项目符号清单：
> "· 苏农 ROA 还可以
> · 中收转化不够
> · 个贷不良需要重视
> · 资本充足率仍有空间"

> **本 Skill 的版式**：
> 左侧深蓝侧栏放大字模块编号（02 / 03 / ...）+ 模块名 + 涵盖页码；右侧 H1 一句话核心结论 + 3 张证据卡（每卡含编号、子标、3 行论证）+ 3 张数字卡（标签、大数字、位次）+ 底部"下一模块预告"过渡条。读者扫一眼就知道这一段的结论 + 证据 + 下一段读什么。

### KPI 摘要页（执行摘要的核心）

> **典型 PPT 模板**：
> 把 4 项指标列成 2×2 表格 + 一段总结文字塞在下面。

> **本 Skill 的版式**：
> 顶部 H1 + H2，下面一条窄"行业锚定条"提供宏观背景，然后并列 4 张 KPI 卡（每卡颜色编码 = 蓝/红/金/红，独立头部颜色条 + 大数字 + 排名标签 + 同业对比 + 一句关键说明），最底部双栏结论（左侧"经营视角"+ 右侧"市场视角"）双视角并立。

## What this skill produces

- **页数**：典型为 40-50 页（开篇 3 + 摘要+方法论 5 + 6-8 个专题模块 + 实证模块 + 矛盾汇总 + 行动地图 + 闭幕 2）
- **画布**：1920×1080，高信息密度独立阅读版
- **预计阅读时长**：30-40 分钟，董事会一次会议读完
- **deliverable**：`<deck>.slides/` 目录含 manifest.json + 共享 `assets/chrome.css` + 每页 HTML
- **风格基调**：RSM 蓝（#1A3A5C 主蓝 + #1565C0 辅蓝）+ 浅灰底，对比色系统化
