# 银行董办对标分析工具下一阶段业务需求说明

## 一、需求背景

当前工具已经完成从静态财报看板到“董办对标分析工作台”的雏形升级，已支持目标银行、对标银行、银行类型和年份选择，并能够生成 VQA 价值质量诊断、专题事实包、图表解释、HTML 演示报告和数据底稿。

下一阶段的核心目标不是继续堆叠更多图表，而是把工具升级为银行董办可以用于正式汇报准备的产品：能够保存项目、复核口径、配置专题判断规则、生成可追溯的 AI 解读，并输出更接近正式董事会材料的报告。

## 二、产品定位

本产品面向银行董事会办公室、战略管理部、财务管理部、资本市场沟通团队，提供“对标选择、经营诊断、专题分析、报告生成、底稿复核”的一体化工具。

产品不定位为单纯数据看板，而是定位为“银行经营对标与董办汇报生成工具”。核心价值包括：

1. 帮助董办快速选择目标银行、对标银行和类型均值参照。
2. 将财务数据转化为经营问题、专题判断和管理动作。
3. 让每条结论都能回到事实包、指标口径和底层数据。
4. 支持生成面向董事会、管理层和资本市场沟通的不同版本报告。

## 三、当前版本基础

当前版本已具备以下能力：

| 模块 | 当前能力 |
| --- | --- |
| 选择口径 | 支持目标银行、对标银行、银行类型、年份、报告版本和对标模板 |
| 项目状态 | 已显示当前项目摘要和四步报告生成流程 |
| 数据层 | 已读取 2020-2025 年银行财务数据，并生成派生指标 |
| VQA诊断 | 已支持盈利真实性、息差防守、风险确认、资本效率、估值验证五维诊断 |
| 专题分析 | 已支持五类专题主体：盈利真实性、息差对冲、风险确认、资本效率、估值验证 |
| AI解读雏形 | 已按董事会版、资本市场版、管理层行动版生成解读草稿 |
| 证据引用 | 已在专题中输出红黄绿判断、证据指标和一句话结论 |
| 报告导出 | 已支持 HTML 演示报告、打印/PDF、选定数据和全部数据导出 |

## 四、路线图判断

基于现有路线图，下一阶段应围绕三条主线继续推进。

### 1. 产品端判断

当前产品已经可以完成单次分析，但还不够像正式工作台。下一阶段需要从“本机临时分析”升级为“可复用项目”，重点解决项目版本、对标组治理、口径复核和报告留痕。

### 2. 展示端判断

当前页面已经具备汇报结构，但导出报告仍需要进一步靠近正式 PPT。下一阶段应形成统一汇报母版，包括目录、章节摘要、专题页、图表页、行动建议页和数据附录页。

### 3. 分析内容端判断

当前专题判断仍主要写在前端逻辑里，下一阶段应配置化。专题的指标权重、红黄绿阈值、必须引用指标、禁用表述和 AI 写稿模板都需要独立配置，避免后续难以维护。

## 五、下一阶段业务目标

下一阶段建议定义为“可复核的董办报告生成版本”。

业务目标如下：

1. 董办可以保存并复用多个分析项目。
2. 每个项目可以记录目标银行、对标银行、年份、类型均值、报告版本、专题选择和图表取舍。
3. 每个专题判断都由可配置规则生成，而不是写死在页面代码中。
4. 每段 AI 解读都引用明确事实包，能够回到指标、年份和图表。
5. 导出的报告具有正式董事会材料的结构，而不是网页截图集合。
6. 数据底稿能够支持财务、战略、董办共同复核。

## 六、用户角色

| 角色 | 使用目标 | 关键诉求 |
| --- | --- | --- |
| 董办人员 | 准备董事会经营分析材料 | 结论清楚、逻辑完整、可快速导出报告 |
| 战略管理部 | 分析同业差距和经营专题 | 对标组合理、专题分析有深度 |
| 财务管理部 | 复核指标口径和底层数据 | 指标来源、计算公式、缺失情况可追溯 |
| 资本市场团队 | 准备投资者沟通材料 | 能区分董事会语言和资本市场语言 |
| 咨询顾问/分析师 | 维护分析框架和报告模板 | 规则可配置，报告可复用 |

## 七、需求范围

### P1-1：项目版本管理

#### 业务说明

当前只能保存最近一个项目。下一阶段需要支持多个分析项目，便于同一银行在不同年份、不同对标组、不同报告版本之间反复调整。

#### 功能需求

| 编号 | 需求 | 说明 |
| --- | --- | --- |
| PRD-01 | 新建分析项目 | 用户可创建一个新的分析项目，填写项目名称 |
| PRD-02 | 保存项目配置 | 保存目标银行、对标银行、年份、银行类型、报告版本、对标模板、当前专题、图表取舍 |
| PRD-03 | 项目列表 | 展示已保存项目列表，包括项目名称、目标银行、年份、更新时间 |
| PRD-04 | 加载项目 | 点击项目后恢复对应分析口径和页面状态 |
| PRD-05 | 复制项目 | 可基于旧项目复制一份，用于新年份或新对标组分析 |
| PRD-06 | 删除项目 | 可删除不再使用的项目 |

#### 验收标准

1. 用户至少可以保存 10 个本地项目。
2. 加载项目后，目标银行、对标银行、年份、报告版本、专题和图表取舍与保存时一致。
3. 项目列表按更新时间倒序展示。

### P1-2：对标组治理

#### 业务说明

当前已支持同类型、同区域、同规模、资本市场相近等对标模板。下一阶段需要让对标组更加可解释、可保存、可复用。

#### 功能需求

| 编号 | 需求 | 说明 |
| --- | --- | --- |
| PRD-07 | 对标推荐理由 | 自动推荐对标银行时，展示推荐逻辑，如同区域、同类型、规模接近、PB/ROA接近 |
| PRD-08 | 对标组保存 | 用户可将当前对标银行保存为自定义对标组 |
| PRD-09 | 对标组命名 | 支持命名为“华东农商行对标组”“上市城商行对标组”等 |
| PRD-10 | 对标组复用 | 新项目可直接选择已保存对标组 |
| PRD-11 | 对标组编辑 | 支持增删银行，并保留最近修改时间 |

#### 验收标准

1. 选择目标银行后，系统自动展示推荐对标组及推荐理由。
2. 用户可以手工调整推荐结果并保存。
3. 已保存对标组可在新项目中复用。

### P1-3：专题判断规则配置化

#### 业务说明

当前专题红黄绿判断已经可用，但规则仍写在前端逻辑里。下一阶段需要将专题指标、权重、阈值和引用规则放到配置文件，便于业务人员调整。

#### 功能需求

| 编号 | 需求 | 说明 |
| --- | --- | --- |
| PRD-12 | 专题配置文件 | 新增专题配置，覆盖盈利真实性、息差对冲、风险确认、资本效率、估值验证 |
| PRD-13 | 指标权重配置 | 每个专题可配置核心指标、方向、权重 |
| PRD-14 | 红黄绿阈值配置 | 每个专题可配置红色、黄色、绿色判断阈值 |
| PRD-15 | 必须引用指标 | 每个专题可配置 AI 解读必须引用的关键指标 |
| PRD-16 | 禁用表述 | 配置不能出现在客户报告中的表述，如“必然改善”“明确低估”等 |
| PRD-17 | 配置生效 | 页面专题判断、AI 解读和导出报告均读取配置 |

#### 验收标准

1. 修改专题配置后，无需改页面代码即可改变专题判断。
2. 每个专题至少包含 3 个核心指标。
3. 红黄绿判断在网页、HTML报告和数据底稿中保持一致。

### P1-4：指标口径复核

#### 业务说明

董办报告需要经得起财务和战略部门复核。下一阶段需要让每个指标展示来源字段、计算公式、可用年份、缺失情况和是否为派生指标。

#### 功能需求

| 编号 | 需求 | 说明 |
| --- | --- | --- |
| PRD-18 | 指标详情弹层 | 点击指标名称后展示指标口径 |
| PRD-19 | 来源字段展示 | 展示指标来自原始 Excel 哪些字段 |
| PRD-20 | 计算公式展示 | 派生指标展示计算公式 |
| PRD-21 | 覆盖情况展示 | 展示当前目标银行和对标银行该指标的年份覆盖 |
| PRD-22 | 缺失原因说明 | 对缺失字段显示“原始数据缺失”“无法计算”“口径不适用”等 |
| PRD-23 | 口径导出 | 数据底稿中增加“指标口径”sheet |

#### 验收标准

1. 专题事实包里的每个指标都能查看口径。
2. 数据覆盖页可下钻到指标详情。
3. 导出的数据底稿包含指标口径、公式和覆盖情况。

### P1-5：AI写稿接口与事实包输入

#### 业务说明

当前 AI 解读是本地框架生成。下一阶段可以接入真正的大模型，但必须遵循“AI只读事实包，不直接编数”的原则。

#### 功能需求

| 编号 | 需求 | 说明 |
| --- | --- | --- |
| PRD-24 | 事实包生成 | 为每个专题和每张图生成结构化事实包 |
| PRD-25 | Prompt模板 | 建立董事会版、资本市场版、管理层行动版三类提示词 |
| PRD-26 | AI解读生成 | 基于事实包生成自然语言解读 |
| PRD-27 | 引用指标标记 | 每段AI解读展示引用了哪些指标 |
| PRD-28 | 重新生成 | 用户可以对单个专题或单张图重新生成解读 |
| PRD-29 | 人工编辑 | 用户可编辑AI解读，并将编辑后的文字进入报告导出 |

#### 验收标准

1. AI 解读中出现的数字必须来自事实包。
2. 每段解读至少展示 2 个引用指标。
3. 用户编辑后的文字可以进入 HTML/PDF/PPTX 导出。

### P1-6：HTML报告母版升级

#### 业务说明

当前已支持 HTML 演示报告，但还需要进一步向正式 PPT 汇报结构靠齐。

#### 功能需求

| 编号 | 需求 | 说明 |
| --- | --- | --- |
| PRD-30 | 报告目录页 | HTML报告增加目录页 |
| PRD-31 | 章节摘要页 | 每章前增加章节摘要，说明本章回答的问题 |
| PRD-32 | 专题主体页 | 每个入选专题生成专题页，包含判断、证据、事实包和AI解读 |
| PRD-33 | 图表页版式 | 每张图固定输出主标题、副标题、图表、关键结论、机制解释、行动启示 |
| PRD-34 | 行动建议页 | 把建议拆成现在、近期、中期三类 |
| PRD-35 | 数据附录页 | 报告末尾增加数据覆盖和指标口径附录 |

#### 验收标准

1. HTML报告至少包含封面、目录、故事线、专题页、图表页、行动页、附录页。
2. 每张图表页在一页内完整显示。
3. 报告文字可以直接用于客户董事会材料初稿。

### P1-7：PPTX可编辑导出

#### 业务说明

董办最终需要可编辑PPT。下一阶段应在 HTML 中间稿稳定后，新增 PPTX 导出能力。

#### 功能需求

| 编号 | 需求 | 说明 |
| --- | --- | --- |
| PRD-36 | PPTX导出入口 | 页面新增“导出PPTX”按钮 |
| PRD-37 | PPT母版样式 | 保持与苏农汇报材料一致的蓝色系、标题、副标题、页脚和品牌样式 |
| PRD-38 | 图表转图片 | 前端SVG图表可进入PPT页面 |
| PRD-39 | 文本可编辑 | 标题、结论、解读和行动建议在PPT中可编辑 |
| PRD-40 | 章节结构一致 | PPTX页序与HTML报告一致 |

#### 验收标准

1. 导出的 PPTX 可以用 PowerPoint 正常打开。
2. 文本内容可编辑。
3. 页数和章节顺序与 HTML 报告一致。

## 八、优先级排序

下一阶段建议按以下顺序推进：

| 优先级 | 模块 | 原因 |
| --- | --- | --- |
| P1-1 | 专题判断规则配置化 | 先把分析规则从页面代码中抽出来，后续才好维护 |
| P1-2 | 指标口径复核 | 董办报告必须可复核，这是正式交付的底座 |
| P1-3 | HTML报告母版升级 | 先把导出中间稿做稳，再进入 PPTX |
| P1-4 | 项目版本管理 | 支持多项目后，产品才像真正工作台 |
| P1-5 | AI写稿接口 | 在事实包稳定后接入，避免AI无依据写稿 |
| P1-6 | PPTX可编辑导出 | 最终交付能力，但建议放在HTML母版稳定之后 |
| P1-7 | 对标组治理 | 可并行推进，提升选择效率和客户解释力 |

## 九、关键业务流程

### 流程一：董办生成一份分析报告

1. 用户新建分析项目。
2. 选择目标银行、分析年份、银行类型。
3. 系统推荐对标银行，并展示推荐理由。
4. 用户调整并保存对标组。
5. 系统生成 VQA 诊断、专题判断和图表分析。
6. 用户选择进入报告的专题和图表。
7. 用户检查AI解读和证据引用。
8. 用户导出 HTML 报告、PDF 或 PPTX。
9. 用户导出数据底稿用于内部复核。

### 流程二：财务部门复核指标口径

1. 财务人员进入数据覆盖或专题事实包。
2. 点击指标名称。
3. 查看来源字段、计算公式、可用年份和缺失原因。
4. 导出口径底稿。
5. 对有问题字段进行标注或反馈。

### 流程三：资本市场团队生成沟通话术

1. 用户选择资本市场沟通版报告。
2. 系统保留估值验证、盈利质量、资本效率等相关专题。
3. AI基于事实包生成资本市场版解读。
4. 用户编辑话术。
5. 导出资本市场沟通材料。

## 十、数据与分析要求

### 1. 事实包字段要求

每个专题和每张图的事实包至少包含：

| 字段 | 说明 |
| --- | --- |
| 目标银行 | 当前分析银行 |
| 对标银行 | 当前对标组 |
| 分析年份 | 当前截面年份 |
| 指标名称 | 中文指标名称 |
| 目标值 | 目标银行当前值 |
| 对标均值 | 对标银行均值 |
| 类型均值 | 所选银行类型均值 |
| 一年变化 | 当前年较上一年变化 |
| 五年变化 | 当前年较2020年或可用最早年份变化 |
| 全样本分位 | 在当前年份样本中的分位 |
| 数据完整性 | 目标银行和对标银行该指标的可用率 |
| 解释方向 | 指标偏高/偏低的业务含义 |

### 2. AI解读约束

AI生成文字必须遵循以下规则：

1. 不允许生成事实包之外的数字。
2. 不允许使用“必然改善”“明确低估”“保证修复”等绝对化表达。
3. 每段结论必须引用至少两个指标。
4. 需要区分董事会版、资本市场版、管理层行动版。
5. 需要说明“为什么”“说明什么”“下一步做什么”。

## 十一、验收指标

| 类型 | 指标 | 验收标准 |
| --- | --- | --- |
| 产品完整性 | 项目可复用 | 保存项目后重新打开能恢复配置 |
| 数据可信度 | 指标可追溯 | 专题事实包指标均可查看口径 |
| 分析可信度 | 解读有引用 | AI解读均显示引用指标 |
| 报告可用性 | HTML报告完整 | 包含封面、目录、专题页、图表页、行动页、附录页 |
| 交付效率 | 报告生成时间 | 选择口径后 30 秒内生成页面报告 |
| 可编辑性 | PPTX可编辑 | 导出的 PPTX 标题和正文可编辑 |

## 十二、非功能需求

| 类型 | 要求 |
| --- | --- |
| 性能 | 页面分析生成应在本地数据量下保持流畅，切换专题不应明显卡顿 |
| 可维护性 | 专题规则、指标口径、报告版本、禁用表述应配置化 |
| 可解释性 | 所有图表和AI解读都应能追溯到事实包 |
| 一致性 | 网页、HTML报告、PDF/PPTX和数据底稿中的结论口径应一致 |
| 中文化 | 页面、字段、报告和导出底稿应避免英文裸字段 |
| 合规性 | 输出内容应避免投资建议式表达和确定性收益表述 |

## 十三、风险与待确认事项

| 风险 | 说明 | 建议 |
| --- | --- | --- |
| 数据口径不完整 | 部分银行或年份字段缺失，可能影响专题判断 | 在事实包中展示完整性，并允许降级生成 |
| AI解读失真 | 如果AI直接生成数字，可能出现不可复核结论 | 强制AI只读取事实包 |
| PPTX导出复杂 | 图表、样式和可编辑文本同时满足有一定工程复杂度 | 先稳定HTML母版，再做PPTX |
| 对标组争议 | 不同客户对“合理对标”的理解不同 | 展示推荐理由，并允许手工保存 |
| 规则维护成本 | 专题越来越多后，硬编码难维护 | 尽快规则配置化 |

## 十四、建议开发里程碑

### 里程碑一：配置化与复核底座

交付内容：

1. 专题规则配置文件。
2. 指标口径详情。
3. 数据底稿新增指标口径 sheet。

## 十七、专题规则配置化PRD

### 1. 业务问题

当前专题判断已经能生成红黄绿信号，但如果指标、权重、阈值、引用指标继续写在页面逻辑里，后续新增专题或调整分析口径会越来越难维护。银行客户也会要求解释：为什么这个专题是红色，为什么引用这些指标，为什么这个指标高低代表好坏。

因此，需要将专题判断规则沉淀为配置化能力。

### 2. 功能需求

| 编号 | 需求名称 | 需求描述 | 优先级 |
| --- | --- | --- | --- |
| PRD-41 | 专题配置读取 | 页面启动时读取专题配置，生成专题列表、专题问题、机制解释和行动建议 | P1 |
| PRD-42 | 专题指标配置 | 每个专题支持配置指标代码、中文名称、方向、权重、是否必须引用 | P1 |
| PRD-43 | 红黄绿阈值配置 | 每个专题支持配置红色、黄色、绿色判断条件 | P1 |
| PRD-44 | 证据指标选择 | 系统根据配置自动选择证据指标，优先选择必须引用指标和低分位指标 | P1 |
| PRD-45 | 禁用表述过滤 | 生成解读时自动检查禁用表述，避免出现确定性投资建议语言 | P1 |
| PRD-46 | 配置异常降级 | 配置缺失或字段不可用时，页面不报错，降级为默认规则 | P1 |
| PRD-47 | 配置版本展示 | 页面展示当前规则版本，便于报告留痕 | P2 |

### 3. 专题配置字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | 文本 | 专题唯一标识 |
| title | 文本 | 专题中文名称 |
| question | 文本 | 董事会需要回答的问题 |
| mechanism | 文本 | 专题背后的经营机理 |
| actions | 数组 | 管理动作建议 |
| metrics | 数组 | 专题指标列表 |
| requiredCitations | 数组 | AI解读必须引用的指标 |
| forbiddenPhrases | 数组 | 禁用表述 |
| thresholds | 对象 | 红黄绿阈值 |

### 4. 业务规则

1. 若一个专题内低分位指标数量达到红色阈值，则专题判断为红色。
2. 若未达到红色但存在低分位指标，或专题平均分低于黄色阈值，则专题判断为黄色。
3. 若强势指标数量达到绿色阈值，且不存在关键红色指标，则专题判断为绿色。
4. 必须引用指标优先进入证据指标区。
5. 若必须引用指标缺失，则用同专题内可用指标替代，并在事实包中标记“引用指标降级”。

### 5. 页面表现

专题页应展示：

1. 专题名称。
2. 董事会问题。
3. 红黄绿专题判断。
4. 一句话结论。
5. 证据指标。
6. 指标事实表。
7. 董事会版解读。
8. 资本市场版解读。
9. 管理层行动版解读。

### 6. 验收标准

1. 修改配置文件中的指标权重后，专题判断结果随之变化。
2. 修改禁用表述后，生成解读不出现对应词语。
3. 删除某个专题配置后，页面仍可加载其他专题。
4. 每个专题至少展示 3 个证据指标。
5. 网页、HTML报告、数据底稿中的专题判断保持一致。

## 十八、指标口径复核PRD

### 1. 业务问题

银行董办报告需要经过财务和战略部门复核。当前页面虽然能展示指标和图表，但用户需要进一步知道指标从哪里来、怎么计算、哪些年份可用、为什么有些字段为空。

因此，需要把指标口径复核做成一个可下钻的业务能力。

### 2. 功能需求

| 编号 | 需求名称 | 需求描述 | 优先级 |
| --- | --- | --- | --- |
| PRD-48 | 指标详情入口 | 专题事实表、图表事实包、数据覆盖字段列表中的指标名称可点击 | P1 |
| PRD-49 | 指标基础信息 | 展示指标中文名、所属主题、指标方向、单位、是否派生指标 | P1 |
| PRD-50 | 来源字段说明 | 展示原始字段名称、来源表、来源年份范围 | P1 |
| PRD-51 | 计算公式说明 | 派生指标展示公式、分子、分母、计算逻辑 | P1 |
| PRD-52 | 当前样本覆盖 | 展示目标银行、对标银行、类型均值在该指标上的覆盖情况 | P1 |
| PRD-53 | 缺失原因展示 | 对不可用指标展示缺失原因，如原始字段缺失、分母为零、口径不适用 | P1 |
| PRD-54 | 年度趋势入口 | 在指标详情中支持跳转到该指标年度趋势图 | P1 |
| PRD-55 | 口径底稿导出 | 导出数据底稿时新增指标口径sheet | P1 |

### 3. 指标详情信息结构

| 信息块 | 展示内容 |
| --- | --- |
| 指标概览 | 指标名称、主题、方向、单位、指标类型 |
| 业务含义 | 指标用于回答什么经营问题 |
| 计算口径 | 公式、来源字段、计算方式 |
| 数据覆盖 | 目标银行、对标银行、类型均值、全样本覆盖率 |
| 年度可用性 | 2020-2025各年份是否可用 |
| 缺失说明 | 缺失原因和是否影响当前结论 |

### 4. 业务规则

1. 原始指标直接展示来源字段。
2. 派生指标必须展示计算公式。
3. 若指标完整性低于 60%，页面应提示“该指标不宜作为主结论依据”。
4. 若目标银行缺失该指标，则专题判断不得把该指标作为证据指标。
5. 导出报告中可保留简化口径，数据底稿中保留完整口径。

### 5. 验收标准

1. 专题事实表中的指标均可打开详情。
2. 每个详情页至少包含指标名称、业务含义、来源字段、覆盖情况。
3. 派生指标必须展示公式。
4. 数据底稿中新增“指标口径”sheet。
5. 用户能从指标详情跳转到年度趋势。

## 二十一、下一轮开发建议

建议下一轮开发按以下顺序执行：

1. **先做专题规则配置化**：这是分析可维护性的底座，也能验证 `analysis_rules.json` 是否足够承载业务规则。

4. **最后补多项目管理和PPTX**：等报告结构稳定后，再做项目版本和可编辑PPT，避免反复返工。
